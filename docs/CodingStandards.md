# Coding Standards

## General Rules
- Use TypeScript for all source files
- Keep functions small and focused
- Use descriptive names for variables and methods
- Follow consistent indentation and formatting

## Testing Rules
- Write tests that are readable and independent
- Avoid hardcoded values where fixtures or factories can be used
- Assert on meaningful response properties

## File Organization
- Place API logic in src/api
- Place reusable data generation in src/factories
- Place shared interfaces in src/models
